# Uniswap V2 / V3 小白学习文档

这份文档的目标不是让你马上看懂所有合约代码，而是先建立一套清晰的脑内模型：

- Uniswap 解决了什么问题
- V2 为什么能自动定价
- LP 为什么能赚手续费，也为什么会有无常损失
- V3 为什么叫集中流动性
- V3 里的 tick、sqrtPrice、liquidity 到底在描述什么
- 学习和实战时最容易踩哪些坑

一句话总览：

```text
Uniswap V2 = 全价格区间的恒定乘积 AMM
Uniswap V3 = 可自定义价格区间的集中流动性 AMM
```

---

## 1. 先理解 Uniswap 在做什么

### 1.1 传统交易所怎么交易

传统中心化交易所通常使用订单簿：

```text
有人挂买单：我愿意用 2990 USDC 买 1 ETH
有人挂卖单：我愿意用 3010 USDC 卖 1 ETH
交易所负责撮合
```

这种模式依赖大量挂单，也依赖专业做市商不断报价。

### 1.2 Uniswap 怎么交易

Uniswap 不用订单簿，而是用资金池：

```text
ETH/USDC 池子里有 ETH 和 USDC
交易者直接和这个池子交易
价格由池子中两种资产的比例决定
```

这个模式叫 AMM：

```text
AMM = Automated Market Maker = 自动做市商
```

在 Uniswap 里，做市商不是某个公司或机构，而是一条数学曲线和一组合约规则。

## 2. 必备词汇表

### Token

代币。比如 ETH、USDC、UNI、WBTC。

### Pair / Pool

交易池。比如：

```text
ETH/USDC 池
WBTC/ETH 池
UNI/USDC 池
```

在 V2 中通常叫 pair，在 V3 中通常叫 pool。

### Swap

兑换。比如用 USDC 换 ETH。

### LP

Liquidity Provider，流动性提供者。

LP 把资产放入池子，让别人可以交易，并赚取手续费。

### Reserve

池子里的资产余额。

比如某个 V2 池子中：

```text
reserveETH = 100 ETH
reserveUSDC = 300,000 USDC
```

### Slippage

滑点。你实际成交价格和当前显示价格之间的差异。

大额交易会改变池子比例，所以成交均价会变差。

### Impermanent Loss

无常损失。LP 相比单纯持币可能少赚或多亏的部分。

## 3. Uniswap V2 核心原理

## 3.1 恒定乘积公式

Uniswap V2 的核心公式是：

```text
x * y = k
```

其中：

```text
x = 池子里 token0 的数量
y = 池子里 token1 的数量
k = 常数
```

例子：

```text
池子里有 100 ETH 和 300,000 USDC

x = 100
y = 300,000
k = 100 * 300,000 = 30,000,000
```

当有人用 USDC 买 ETH 时：

```text
USDC 进入池子，所以 y 增加
ETH 离开池子，所以 x 减少
为了让 x * y 维持在同一条曲线上，ETH 价格会上升
```

这就是 V2 自动定价的来源。

---

## 3.2 价格从哪里来

如果池子里有：

```text
100 ETH
300,000 USDC
```

那么池子隐含价格是：

```text
ETH 价格 = USDC 数量 / ETH 数量
         = 300,000 / 100
         = 3,000 USDC
```

一般公式：

```text
token0 用 token1 计价 = reserve1 / reserve0
token1 用 token0 计价 = reserve0 / reserve1
```

注意：这个是边际价格，也就是非常小的一笔交易面对的价格。实际大额交易会有滑点。

---

## 3.3 Swap 是怎么计算的

假设池子中有：

```text
100 ETH
300,000 USDC
```

用户想用 3,000 USDC 买 ETH。先忽略手续费。

交易后 USDC 数量变成：

```text
300,000 + 3,000 = 303,000
```

为了保持：

```text
x * y = 30,000,000
```

新的 ETH 数量应为：

```text
newETH = 30,000,000 / 303,000
       = 99.00990099
```

所以用户拿到的 ETH：

```text
100 - 99.00990099 = 0.99009901 ETH
```

你可能会发现：

```text
3,000 USDC 并没有买到完整 1 ETH
```

这就是滑点。买得越多，成交均价越差。

---

## 3.4 V2 手续费

Uniswap V2 默认交易手续费是：

```text
0.30%
```

如果用户输入 3,000 USDC，真正进入定价公式的不是 3,000，而是：

```text
3,000 * 0.997 = 2,991 USDC
```

V2 常见输出公式：

```text
amountInWithFee = amountIn * 997

amountOut =
  amountInWithFee * reserveOut
  /
  (reserveIn * 1000 + amountInWithFee)
```

手续费留在池子里，最终归 LP 所有。

---

## 3.5 为什么池子越大滑点越小

同样用 3,000 USDC 买 ETH：

小池子：

```text
100 ETH / 300,000 USDC
```

这笔交易相当于池子 USDC 的 1%。

大池子：

```text
10,000 ETH / 30,000,000 USDC
```

这笔交易只相当于池子 USDC 的 0.01%。

所以：

```text
池子越深，交易对比例扰动越小，滑点越低
```

## 4. V2 LP 是怎么赚钱的

### 4.1 LP 如何提供流动性

在 V2 中，LP 需要按当前池子比例同时存入两种资产。

如果当前池子比例是：

```text
1 ETH = 3,000 USDC
```

那么你想存入 1 ETH，就大约还需要存入 3,000 USDC。

你不能只存 ETH 或只存 USDC，因为 V2 池子需要维持两边资产比例。

---

### 4.2 LP Token

存入资产后，LP 会收到 LP Token。

LP Token 代表你对池子的份额。

例子：

```text
池子总价值 = 1,000,000 USDC
你存入价值 = 100,000 USDC

你拥有池子 10% 份额
```

之后如果有人交易，手续费留在池子里，池子变大。你取回资产时，按份额拿回。

---

### 4.3 LP 收益来源

LP 的收益主要来自：

```text
交易手续费
```

但 LP 的资产组合会随着价格变化自动改变。

价格上涨时，池子会逐渐卖出上涨资产；
价格下跌时，池子会逐渐买入下跌资产。

这就是 LP 的本质：

```text
你在用一个自动再平衡策略做市
```

---

## 5. 无常损失 Impermanent Loss

### 5.1 先用直觉理解

假设你在 ETH = 3,000 USDC 时提供流动性：

```text
1 ETH + 3,000 USDC
```

如果 ETH 后来涨到 6,000 USDC：

单纯持币会变成：

```text
1 ETH = 6,000 USDC
3,000 USDC = 3,000 USDC
总价值 = 9,000 USDC
```

但如果你是 LP，池子会在 ETH 上涨过程中不断卖出 ETH、买入 USDC。

所以最后你持有的 ETH 会少于 1 个，USDC 会多于 3,000。

你仍然赚钱，但通常会比单纯持币少赚。

这个少赚的部分就是无常损失。

---

### 5.2 无常损失公式

假设价格变化倍数是：

```text
r = 新价格 / 旧价格
```

V2 无常损失公式：

```text
IL = 2 * sqrt(r) / (1 + r) - 1
```

例子：价格涨 2 倍

```text
r = 2
IL = 2 * sqrt(2) / 3 - 1
   ≈ -5.72%
```

例子：价格涨 4 倍

```text
r = 4
IL = 2 * sqrt(4) / 5 - 1
   = 4 / 5 - 1
   = -20%
```

价格下跌也一样。

比如价格跌到原来的 1/4：

```text
r = 0.25
IL = 2 * sqrt(0.25) / 1.25 - 1
   = 1 / 1.25 - 1
   = -20%
```

---

### 5.3 为什么叫“无常”

因为如果价格回到你入场时的价格，无常损失会消失。

但如果你在价格偏离时退出，这个损失就变成实际损失。

所以更准确的理解是：

```text
无常损失 = LP 策略相对于持币策略的相对损益
```

它不是绝对亏损，也不是一定亏钱。

如果手续费足够高，LP 总收益仍然可能超过持币。

## 6. V2 的优点和缺点

### 优点

```text
简单
被动
任何价格都能交易
LP 不需要选择价格区间
```

### 缺点

```text
资金效率低
大量流动性分布在几乎不会成交的极端价格区间
LP 无法表达自己的价格判断
所有交易对基本使用同一种做市曲线
```

这就引出了 V3。

---

## 7. Uniswap V3 为什么出现

V2 的流动性分布在：

```text
0 到 正无穷
```

但大部分交易其实发生在当前价格附近。

比如 ETH 当前价格是 3,000 USDC，大部分交易可能发生在：

```text
2,800 - 3,200
```

很少有人会在短时间内用：

```text
100 USDC 买 ETH
或
100,000 USDC 买 ETH
```

所以 V2 把资金平均铺在整条价格曲线上，会造成大量资金闲置。

V3 的核心改进：

```text
LP 可以自己选择做市价格区间
```

这叫集中流动性。

---

## 8. V3 集中流动性

### 8.1 一个简单例子

假设 ETH 当前价格是 3,000 USDC。

V2 LP：

```text
在 0 到 无限大的所有价格都提供流动性
```

V3 LP 可以选择：

```text
只在 2,700 - 3,300 提供流动性
```

这样同样的资金会集中在更小区间里。

结果：

```text
当前价格附近流动性更深
交易者滑点更低
LP 资金效率更高
```

---

### 8.2 价格在区间内和区间外

假设你的 V3 区间是：

```text
2,700 - 3,300 USDC/ETH
```

如果当前价格在区间内：

```text
你同时持有 ETH 和 USDC
你正在提供流动性
你可以赚手续费
```

如果价格低于 2,700：

```text
你的仓位基本变成 ETH
不再提供有效流动性
不再赚这个区间里的交易手续费
```

如果价格高于 3,300：

```text
你的仓位基本变成 USDC
不再提供有效流动性
不再赚这个区间里的交易手续费
```

直觉：

```text
价格向上走，LP 不断卖出 ETH，换成 USDC
价格向下走，LP 不断买入 ETH，卖出 USDC
```

V3 的 LP 更像是在某个价格区间里挂了一个自动网格策略。

## 9. V3 的价格、tick 和 sqrtPrice

### 9.1 为什么需要 tick

V3 不是让 LP 输入任意小数价格，而是把价格分成一个个离散格子。

这些格子叫 tick。

tick 和价格的关系：

```text
price = 1.0001 ^ tick
```

反过来：

```text
tick = log(price) / log(1.0001)
```

所以 tick 本质上是价格的编号。

---

### 9.2 tickSpacing

不同手续费池有不同的 tick 间隔。

比如某些池子可能只能每隔 10、60、200 个 tick 设置区间边界。

原因是：

```text
低手续费池通常适合稳定资产，需要更细价格粒度
高手续费池通常适合高波动资产，可以使用更粗价格粒度
```

---

### 9.3 为什么 V3 使用 sqrtPrice

V3 合约里经常看到：

```text
sqrtPriceX96
```

含义是：

```text
sqrtPriceX96 = sqrt(price) * 2^96
```

为什么不用 price，而用 sqrt(price)？

因为在 V3 的数学里，用平方根价格可以让 token0、token1 和 liquidity 的计算更简洁，也更适合整数计算。

为什么乘以 2^96？

因为 EVM 不擅长浮点数，链上通常用整数模拟小数。

```text
X96 = 固定点数格式，表示这个数放大了 2^96 倍
```

---

## 10. V3 的 liquidity L

V2 里我们经常看：

```text
x * y = k
```

V3 里更常看到：

```text
L = liquidity
```

L 可以理解为：

```text
在某个价格区间内，你提供的有效流动性深度
```

### 10.1 区间内的资产数量公式

设：

```text
P  = 当前价格
Pa = 区间下界价格
Pb = 区间上界价格
L  = 流动性
```

当价格在区间内时：

```text
token0 数量 = L * (1 / sqrt(P) - 1 / sqrt(Pb))
token1 数量 = L * (sqrt(P) - sqrt(Pa))
```

直觉：

```text
价格越接近下界，token0 越多，token1 越少
价格越接近上界，token0 越少，token1 越多
```

如果 token0 是 ETH，token1 是 USDC：

```text
ETH 价格上涨时，仓位逐渐从 ETH 变成 USDC
ETH 价格下跌时，仓位逐渐从 USDC 变成 ETH
```

---

### 10.2 区间外的资产状态

如果：

```text
P <= Pa
```

仓位全部是 token0。

公式近似：

```text
token0 = L * (1 / sqrt(Pa) - 1 / sqrt(Pb))
token1 = 0
```

如果：

```text
P >= Pb
```

仓位全部是 token1。

公式近似：

```text
token0 = 0
token1 = L * (sqrt(Pb) - sqrt(Pa))
```

## 11. V3 为什么资金效率更高

假设当前 ETH 价格是 3,000 USDC。

V2 把资金放在：

```text
0 到 无限大
```

V3 如果只放在：

```text
2,700 到 3,300
```

那么同样 10,000 USDC 的资金，可以在这个局部区间提供更深的流动性。

资金效率提升来自：

```text
更少的价格范围
更集中的资金
更高的局部深度
```

但代价是：

```text
如果价格跑出区间，就不再赚手续费
并且仓位会变成单边资产
```

所以 V3 不是单纯“更好”，而是：

```text
更高效率 + 更高管理要求
```

## 12. V3 手续费等级

V3 支持多个手续费等级。

常见等级包括：

```text
0.01%
0.05%
0.30%
1.00%
```

不同资产适合不同费率：

```text
稳定币对：通常适合低费率
高相关资产：适合低到中等费率
普通波动资产：常见 0.30%
长尾高波动资产：可能使用更高费率
```

选择手续费等级，本质是在权衡：

```text
低手续费 = 更容易吸引交易量
高手续费 = 每笔交易 LP 收入更高
```

实际哪个池子最好，要看：

```text
交易量
流动性深度
竞争池子
价格波动
LP 区间设置
```

## 13. V2 和 V3 的核心差异

| 维度 | Uniswap V2 | Uniswap V3 |
|---|---|---|
| 流动性范围 | 全价格区间 | 自定义价格区间 |
| 资金效率 | 较低 | 较高 |
| LP 难度 | 简单 | 更复杂 |
| LP 仓位表示 | ERC-20 LP Token | NFT 仓位 |
| 手续费 | 主要是 0.30% | 多档费率 |
| 是否需要主动管理 | 较少 | 通常需要 |
| 适合人群 | 被动 LP、新手理解 AMM | 专业 LP、主动策略 |

## 14. V3 LP 策略直觉

### 14.1 宽区间

例如：

```text
1,500 - 6,000 USDC/ETH
```

特点：

```text
更像 V2
不容易出区间
资金效率较低
手续费收益较稳定
管理频率较低
```

### 14.2 窄区间

例如：

```text
2,900 - 3,100 USDC/ETH
```

特点：

```text
资金效率高
如果交易活跃，手续费收益可能很高
容易出区间
需要频繁调整
更依赖价格判断
```

### 14.3 单边区间

有些 LP 会故意把区间设置在当前价格一侧。

这类似于：

```text
用限价单买入
或
用限价单卖出
```

例如当前 ETH = 3,000：

```text
设置 2,700 - 2,900 的区间
```

如果价格跌进去，你会逐渐把 USDC 换成 ETH。

这有点像：

```text
我愿意在更低价格分批买 ETH
```

## 15. 一个完整例子：从 V2 到 V3

### 场景

你有：

```text
1 ETH
3,000 USDC
```

ETH 当前价格：

```text
3,000 USDC
```

### 如果你在 V2 做 LP

你把：

```text
1 ETH + 3,000 USDC
```

放进 ETH/USDC 池子。

你会在所有价格上做市。

好处：

```text
不用选区间
只要池子有交易，就能按份额赚手续费
```

坏处：

```text
资金效率低
ETH 大涨或大跌时会有明显无常损失
```

### 如果你在 V3 做 LP

你选择区间：

```text
2,700 - 3,300
```

只要价格在区间内：

```text
你提供有效流动性
你赚手续费
```

如果价格涨到 3,500：

```text
你基本变成 USDC
停止赚这个区间的手续费
如果 ETH 继续涨，你会跑输持有 ETH
```

如果价格跌到 2,500：

```text
你基本变成 ETH
停止赚这个区间的手续费
如果 ETH 继续跌，你承担 ETH 下跌风险
```

## 17. 常见误区

### 误区 1：LP 一定赚钱

不一定。

LP 会赚手续费，但也承担：

```text
无常损失
价格波动风险
合约风险
区间管理风险
```

### 误区 2：V3 一定比 V2 好

不一定。

V3 资金效率高，但需要选区间。

如果区间选错：

```text
赚不到手续费
资产变成单边
频繁调仓还会付 gas 和承受滑点
```

### 误区 3：无常损失是“亏钱”

不完全是。

无常损失是相对于持币的表现差异。

你可能：

```text
绝对收益为正
但相对单纯持币少赚
```

### 误区 4：V3 出区间就是爆仓

不是。

出区间不是爆仓，也不是强平。

它只是意味着：

```text
你的仓位变成单边资产
暂时不再提供有效流动性
暂时不赚该区间手续费
```

```text
price_token1_in_token0 = reserve0 / reserve1
```

```text
amountInWithFee = amountIn * 997

amountOut =
  amountInWithFee * reserveOut
  /
  (reserveIn * 1000 + amountInWithFee)
```

### V2 无常损失

```text
IL = 2 * sqrt(r) / (1 + r) - 1
```

### V3 tick 和价格

```text
price = 1.0001 ^ tick
tick = log(price) / log(1.0001)
```

### V3 sqrtPriceX96

```text
sqrtPriceX96 = sqrt(price) * 2^96
```

### V3 区间内资产数量

```text
amount0 = L * (1 / sqrt(P) - 1 / sqrt(Pb))
amount1 = L * (sqrt(P) - sqrt(Pa))
```

## 20. 小练习

### 练习 1：计算价格

池子里有：

```text
50 ETH
150,000 USDC
```

问题：

```text
ETH 当前价格是多少？
```

答案：

```text
150,000 / 50 = 3,000 USDC
```

### 练习 2：理解滑点

同样价格下，两个池子：

```text
池子 A：10 ETH / 30,000 USDC
池子 B：1,000 ETH / 3,000,000 USDC
```

如果你用 3,000 USDC 买 ETH，哪个池子滑点更低？

答案：

```text
池子 B
```

因为 3,000 USDC 只占池子 B 的很小比例。

```text
不在赚这个区间的手续费
仓位主要变成 token1
```

## 21. 用一句话总结

Uniswap V2 的核心是：

```text
用 x * y = k 这条曲线自动定价
```

Uniswap V3 的核心是：

```text
让 LP 把资金集中放到自己选择的价格区间里
```

LP 的核心收益和风险是：

```text
收益 = 手续费
风险 = 无常损失 + 价格风险 + 区间管理风险 + 合约风险
```

真正理解 Uniswap 的关键不是背公式，而是理解：

```text
交易者改变池子比例
池子比例决定价格
LP 承担库存变化
手续费是 LP 承担这种风险的补偿
```
